1) максимально разбивать css на разные файлы
2) в первую очередь отдельные css файлы для переменных и шрифтов
3) медиа запросы тоже в отдельном файле
4) порядок подключения: шрифты, переменные, осн. файлы, медиа
5)  наследуются: 
    color,font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space и word-spacing 
6) не наследуются:  
    background, border, display, float и clear, height и width, margin, min-max-height и -width, outline, overflow, padding, position,text-decoration, vertical-align и z-index
7) *     0001 - приоритетность
   <tag> 0010
   class 0100
   #id   1000
8)  #id class class tag                 (1120)
    #id class tag tag - приоритетнее    (1210)
9) h1 + p — выберет все первые абзацы, идущие непосредственно за любым тегом <h1>, не затрагивая остальные абзацы
10) p > strong — выберет все элементы strong, являющиеся дочерними по отношению к элементу p.
11) h1 ~ p — выберет все абзацы, являющиеся сестринскими по отношению к любому заголовку h1 и идущие сразу после него
12) :: - псевдоэлемент (before, after...), : - псевдокласс (hover, active...)
13) для указания размеров блока- указывать только ширину и внутренние отступы, остальная высота зависи от кол-ва контента
